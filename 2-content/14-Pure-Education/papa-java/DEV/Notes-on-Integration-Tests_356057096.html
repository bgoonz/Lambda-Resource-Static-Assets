Process Response with JSONArray

Both the variable and the function call must specify String as in:

ResponseEntity<String> response = testRestTemplate.getForEntity(createUrlWithPort("/employees/subscriptions"), String.class);

Note what would happen if we used following syntax:

ResponseEntity<?> response = testRestTemplate.getForEntity(createUrlWithPort("/employees/subscriptions"), String.class);

When we use:

JSONArray jsonArray = new JSONArray(response.getBody());

we will get following error:

org.json.JSONException: Not a primitive array: class java.lang.String

One-To-Many Relations are Lost

Let's look at EmployeeSubscriptionController.getEmployeesSubscriptions method:

@GetMapping("/employees/{id}/subscriptions")
public ResponseEntity<Resource<Employee>> getEmployeesSubscriptions(@PathVariable("id") int id)
{
    List<EmployeeSubscriptionPair> employeeSubscriptionPairs = employeeSubscriptionMapRepository.findByEmployeeId(id);
    if(employeeSubscriptionPairs.isEmpty())
    {
        throw new NotFoundException("employee-" + id);
    }
    Optional<Employee> employeeOptional = employeeSubscriptionPairs.stream().map(EmployeeSubscriptionPair::getEmployee).findFirst();
    List<Subscription> subscriptions = employeeSubscriptionPairs
            .stream()
            .map(EmployeeSubscriptionPair::getSubscription)
            .collect(Collectors.toList());
    Employee employee = employeeOptional.get();
    Resource<Employee> employeeResource = createEmployeeSubscriptionResource(employee, subscriptions);
    employeeResource.add(linkTo(methodOn(getClass()).getEmployeesSubscriptions()).withRel("all"));
    return ResponseEntity.ok(employeeResource);
}

We can fix this problem by adding @Transient collections to Project, Employee, and Subscription Entities, for example:

..
@Transient
private List<Subscription> subscriptions;
public List<Subscription> getSubscriptions()
{
    return subscriptions;
}
public void setSubscriptions(List<Subscription> subscriptions)
{
    this.subscriptions = subscriptions;
}

Each set of Integration Tests runs successfully. But, when all tests are executed some of them fail.

This must be related to dependency. We need to find a way to reset the in-memory database at the beginning of each test.

Add following annotation to the beginning of each integration test:

@DirtiesContext(classMode = DirtiesContext.ClassMode.BEFORE_EACH_TEST_METHOD)
